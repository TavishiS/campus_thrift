# 🛒 CampusThrift

CampusThrift is a cross-platform (Mobile & Web) marketplace application designed exclusively for college and university campuses. It allows students to easily buy, sell, and thrift everyday items like books, electronics, and dorm essentials within their local campus community.

## ✨ Salient Features

* **Secure Authentication:** User login and registration system.
* **Real-Time Marketplace Feed:** A live, automatically updating feed of all items currently for sale.
* **Media-Rich Listings:** Users can upload images directly from their gallery to showcase their items, complete with a multi-image swipeable carousel.
* **Smart Filtering & Sorting:** Users can instantly filter items by categories (Books, Electronics, Mattress, etc.) and sort by price (Low to High / High to Low).
* **Native Email Integration:** Buyers can securely contact sellers. The app seamlessly launches the user's native email client (like Gmail) with a pre-filled subject line and recipient address.
* **Inventory Management:** Sellers can easily mark their items as sold, securely deleting the listing and images from the database.
* **Cross-Platform Ready:** Fully responsive design that works beautifully as an Android app, iOS app, and a Web application. (not checked on ios yet...might have issues)

## 🛠 Tech Stack

* **Frontend Framework:** [Flutter](https://flutter.dev/) (Dart)
* **Backend as a Service (BaaS):** Google Firebase
* **Native OS Integration:** `url_launcher` (for deep-linking and `mailto:` intents), `image_picker` (for native gallery access)

## 🔥 The Power of Firebase

Firebase served as the complete serverless backend for CampusThrift, drastically speeding up development time. Its integration was highly useful in the following ways:

1.  **Firebase Authentication:** Handled complex user management, password encryption, and secure sessions out-of-the-box without needing to build custom login APIs.
2.  **Cloud Firestore:** Acted as the NoSQL database. Using Flutter's `StreamBuilder` paired with Firestore, the app listens to the database in *real-time*. When a user adds or deletes an item, the marketplace feed updates instantly for all users without needing to refresh the page.
3.  **Firebase Cloud Storage:** Provided a robust and secure way to upload, host, and retrieve heavy image files generated by users.
4.  **Firebase Hosting:** Allowed for single-command deployments to host the web version of the application on a secure, global CDN.

## 🚀 Deployment & Sharing Guide

Whenever you make modifications to the app and want to push the updates to your friends, follow these concise steps.

### Part 1: Deploying to the Web

1. **Compile for the web:**
   ```bash
   flutter build web
   ```
2. **Authenticate with Firebase (if not already logged in):**
    ```bash
    firebase login
    ```
3. **Initialize Hosting (Only required the very first time):**
    ```bash
    firebase init hosting
    ```
    * Project: Choose Use an existing project -> select your CampusThrift project.
    * Public directory: Type exactly build/web (Do not skip this!).
    * Single-page app: y (Yes).
    * GitHub automatic builds: N (No).
    * Overwrite index.html: N (No).

4. **Deploy to the internet:**

```bash
firebase deploy --only hosting
```
Firebase will output your live URL (e.g., https://your-project.web.app). Share this link!

### Part 2: Building the Android APK

1. **Clean and fetch fresh dependencies:**

    ```bash
    flutter clean
    flutter pub get
    ```
2. **Build the highly optimized release APK:**

    ```bash
    flutter build apk --release
    ```
3. **Share with friends:**

    Locate the generated file here: build/app/outputs/flutter-apk/app-release.apk

    Send the .apk file directly over any message sharing platform.
